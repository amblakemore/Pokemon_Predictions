Sara McLean: HTML Website Framework

The main goal for developing the project 3 website was to create a useful framework for the team to display the predictive modeling demonstration and results.

The secondary goal was to create a design-forward website keeping in line with current company website. Relying heavily on Bootstrap, along with further work on CSS and learning more about design features such as flat icons, animated images, homepage rotating carousels, I used what I learned to create the framework the viewer ultimately experiences on this site.

Ultimately, I learned I enjoy front-end web design and look forward to continuing to learn more.

Aaron Mahalik: Heroku

Heroku was really easy to use and deploy our web app with. I plan on using it in the future for sure.

I am interested in learning many other cloud based web app deployment services since there are so many to choose from. I love that we could use flask when creating our project, and heroku can basicly install any and all of the packages/dependenies that we used in our scripts.

There were very few requirements that needed to be added to our folder structure/repo. It makes it so deployment is very convenient and quick.

Alex Blakemore: Database Cleaning

In order for image classification to be used to train a model, each class needs labeled example photos. For any project, cleaning data is a critical process. In order to ensure that the model is learning is the most accurate and lead to the best results, we had to go through and manually look at all the images in order to remove images that were not relevant to the model or would cause confusion.

After cleaning the data and then using the developed model, images that were initially removed, maybe should have been used in the training to perhaps assist the model to be more accurate.

If attempting the project again, I would leave more images in the folder and look to add even more images to attempt to improve the model.
